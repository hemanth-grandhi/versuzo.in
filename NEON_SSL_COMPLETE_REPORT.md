## C. POSTGRESQL CONNECTION CONFIGURATION

### Neon Connection String Format

**Required Format:**
```
postgresql://USERNAME:PASSWORD@ep-XXXXXX.neon.tech:5432/DATABASE_NAME
```

**Example (Real):**
```
postgresql://postgres:secretPassword@ep-silent-moon-123456.us-east-1.neon.tech:5432/versuzo_prod
```

### Pool Configuration (Generated by Code)

**Production Environment:**
```typescript
{
  connectionString: "postgresql://user:password@ep-xxxxx.neon.tech:5432/database",
  ssl: {
    rejectUnauthorized: false
  }
}
```

**Development Environment (Local PostgreSQL):**
```typescript
{
  connectionString: "postgresql://user:password@localhost:5432/versuzo_dev",
  // ssl: undefined (not added for local connections)
}
```

### SSL Configuration Explanation

| Setting | Value | Meaning |
|---------|-------|---------|
| `ssl: { }` | Enabled | Activates TLS/SSL encryption |
| `rejectUnauthorized` | `false` | Allows self-signed certificates from Neon |

**Why `rejectUnauthorized: false`?**

- Neon uses self-signed or dynamically-generated SSL certificates
- Standard certificate chain validation would fail
- Setting to `false` disables certificate validation
- **This is SAFE** because Neon is trusted infrastructure (not an untrusted network)
- Connection is still encrypted (SSL provides confidentiality)
- No credentials are exposed

### Environment Auto-Detection

**Code Logic:**
```typescript
if (process.env.NODE_ENV === "production" || validation.host?.includes("neon"))
```

**Enables SSL When:**
1. `NODE_ENV` environment variable is `"production"` → SSL enabled
2. **OR** database hostname contains `"neon"` → SSL enabled

**Does NOT enable SSL When:**
1. `NODE_ENV` is `"development"` **AND** hostname does not contain `"neon"`

**Examples:**
```
Scenario 1: NODE_ENV="production" → SSL enabled ✓
Scenario 2: NODE_ENV="development" + "neon" hostname → SSL enabled ✓
Scenario 3: NODE_ENV="development" + "localhost" → No SSL ✓
```

### Complete Neon Connection Flow

```
1. Backend starts
   ↓
2. DATABASE_URL validation passes
   ↓
3. Hostname parsed: "ep-xxxxx.neon.tech"
   ↓
4. NODE_ENV="production" OR hostname contains "neon"?
   → YES, both conditions true
   ↓
5. poolConfig.ssl = { rejectUnauthorized: false }
   ↓
6. new Pool(poolConfig) with SSL enabled
   ↓
7. pool.connect() establishes TLS handshake
   ↓
8. SSL certificate accepted (rejectUnauthorized=false)
   ↓
9. PostgreSQL connection established over encrypted TLS channel
   ↓
10. ✓ Successfully connected to Postgres database
```
